### 👀 회고
HTML, CSS를 막 떼고 JS를 혼자 공부하다가 실전으로 배우고 싶은 욕심에 시작한 마켓컬리 클론코딩 팀 프로젝트.
뜻이 맞는 INU 사람들이 모여 매주 업무를 할당하고 zoom으로 진행 상황을 체크했다.<br>
🥰 처음 업무가 주어졌을 땐 JS에 J도 모르는 상태에서 시작한 터라 팀원들에게 누가되지 않을까 걱정했지만, 부담감 없이 각자 페이스에 맞춰 진행하기로 했기 때문에 스트레스 받지 않고 끝까지 진행할 수 있었던 것 같다. 처음 시작할 때 팀원마다 다르지만 스스로 순수 JS만을 사용해 구현하는 걸 목표로 했었고 그 과정에서 JS의 원리를 이해하며 실력이 많이 상승했다.<br><br>
🥲 아쉬웠던 점은 깃허브의 project, pull request, issue 등을 적극적으로 사용해 협업 관리를 했으면 더 효율적으로 진행되지 않았을까 싶다. 또한 프론트앤드 업무를 구현하기 위해 백앤드와 소통할 일이 생기는데 백앤드 작동원리를 하나도 알지 못해서 답답했었고 node.js를 배워 백앤드가 어떻게 돌아가는지 알고 싶은 욕심이 생겼다. 또한 링크를 타고 넘어갈 때마다 전체 화면이 리로드되는 것이 불편했고 웹 어플리케이션을 구현하기 위해선 react나 vue 등의 프레임워크를 사용하면 된다는 걸 알았다.<br><br>
3개월 간의 프로젝트는 이렇게 끝이 났지만 혼자만의 온전한 웹 어플리케이션을 구현하는 것을 목적으로 개인 프로젝트를 진행할 예정이다. 아마 typescript와 리액트를 사용할 것 같고 css의 단점을 보완한 scss도 사용하고 싶다. node.js로 백앤드 파트도 혼자 구현해보고 싶지만 지금은 프론트앤드에 집중하자! 

⭐️ 더 자세한 사항은 [e-commerce-clone](https://github.com/e-commerce-clone) 에서 확인해주세요!
